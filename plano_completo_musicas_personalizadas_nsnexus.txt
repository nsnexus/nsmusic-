PLANO COMPLETO — PLATAFORMA DE MÚSICAS PERSONALIZADAS COM INTELIGÊNCIA ARTIFICIAL
Projeto NSNexus

======================================================================
1. VISÃO GERAL DO NEGÓCIO
======================================================================

A proposta é criar uma plataforma de venda de músicas personalizadas, em que o cliente conta uma história, escolhe o estilo musical, aprova a letra, realiza o pagamento e recebe uma música produzida com auxílio de inteligência artificial.

O sistema será parcialmente automatizado no início:

- O cliente preenche as informações.
- A inteligência artificial gera a letra.
- O cliente revisa e aprova.
- O cliente escolhe o pacote e os produtos adicionais.
- O pagamento é realizado.
- O pedido entra no painel administrativo.
- A música é produzida manualmente no Suno.
- Os arquivos são revisados.
- A entrega é feita por uma página privada, WhatsApp e e-mail.

A geração manual no Suno é recomendada no começo porque permite escolher a melhor versão, corrigir problemas de pronúncia e garantir maior qualidade antes da entrega.

9. GERAÇÃO DA LETRA
======================================================================

Prompt base:

Você é um compositor profissional especializado em músicas personalizadas e emocionais.

Crie uma letra original em português brasileiro utilizando somente as informações fornecidas pelo cliente.

Regras:

- Não invente fatos importantes.
- Não copie músicas existentes.
- Não cite artistas.
- Não imite letras famosas.
- Utilize linguagem natural.
- Evite rimas forçadas.
- Inclua o nome naturalmente.
- Inclua a frase obrigatória.
- Evite os assuntos proibidos.
- Crie um refrão marcante.
- Estruture a letra.
- Mantenha duração aproximada de 2 minutos e 30 segundos a 3 minutos e 30 segundos.

Estrutura:

[Verso 1]
[Pré-Refrão]
[Refrão]
[Verso 2]
[Ponte]
[Refrão Final]

Dados:

Estilo:
Emoção:
Ocasião:
Pessoa homenageada:
História:
Momentos importantes:
Qualidades:
Frase obrigatória:
Assuntos proibidos:

Limite recomendado antes do pagamento:

- Uma letra inicial.
- Até duas pequenas alterações.
- Nova composição completa somente mediante pagamento ou pacote.

10. PAGAMENTO
======================================================================

Fluxo:

1. O sistema cria o pedido.
2. O pedido recebe um código.
3. O sistema cria o pagamento.
4. O cliente paga.
5. O Mercado Pago envia um webhook.
6. O servidor valida o webhook.
7. O servidor consulta o pagamento.
8. O valor é comparado com o pedido.
9. O status é atualizado.
10. A produção é liberada.

Nunca libere a produção apenas porque o cliente acessou a página de sucesso.

Dados registrados:

- ID do pagamento.
- Código do pedido.
- Valor.
- Pacote.
- Adicionais.
- Forma de pagamento.
- Status.
- Data de aprovação.
- Dados do webhook.
- Histórico de alterações.
